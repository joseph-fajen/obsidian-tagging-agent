# Project Catalyst — Style Guide

## Voice and Tone
- Professional but approachable
- Technical accuracy over simplicity
- Active voice preferred

## Code Examples
- Always use real, working code
- Include expected output in comments
- Show error cases, not just happy paths

## Terminology
- Use "validator" not "node" for PoS
- Use "transaction" not "tx" in prose
- Capitalize protocol names
