# Documentation Principles

## My Core Beliefs

### 1. User Journey First
Every doc should answer: "What is this?" and "Why should I care?" before "How do I use it?"

### 2. Assume Nothing
If there's an unstated assumption, make it explicit. Developers will thank you.

### 3. Show, Don't Tell
A working code example is worth a thousand words of explanation.

### 4. Test Your Docs
If you can't follow your own getting-started guide and succeed, it's broken.

### 5. Living Documentation
Docs that aren't maintained are worse than no docs. They build false confidence.
