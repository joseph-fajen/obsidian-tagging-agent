# Slack Thread Documentation Idea

Had an idea while reading a Slack thread today: what if we could automatically turn important Slack threads into documentation drafts?

The flow would be:
1. React to a message with a custom emoji
2. n8n webhook picks it up
3. Claude summarizes the thread
4. Creates a draft doc in the repo

This combines #ai-tools with #productivity in a way the team would actually use.
